# Reality Thread 6更新 - 版本0.3.6.2

![news](https://pub-60a5d261178e40e98b04d0c1a4bbcaea.r2.dev/news/screenshot-2026-01-25-014552.png)

2026年1月25日 – 澳大利亚悉尼 – 0.3.6.2版本更新正在推出。本次更新包含一些关键修复、系统改进和重要的后端变更，重点关注稳定性、库存可靠性以及为Reality Thread 7做的准备工作。

虽然许多更改在视觉上并不明显，但它们解决了更深层次的系统问题，这些问题对于长期开发和未来功能发布至关重要。

### 更新详情

- 针对传送后水体检测的额外修复版本，以及针对其他水体检测问题的修复。团队将继续密切监控任何进一步的问题。无忧传送回归！

- 修复了导致制作配方有时失败的问题。

- 上述修复#2导致对大多数现有角色系统进行了进一步的重构。这些更改提高了整体系统可靠性，并更新了系统，为Reality Thread 7做准备。

- 对影响某些角色的隐形腿问题应用了额外的修复。

- 调整了库存逻辑，因此：• 鼠标左键移动单个物品 • 鼠标右键移动所有物品。现在这在所有系统中都是通用的，并且适用于在库存、采集、制作和Mentar存储之间交换物品时。

- 更新了操作栏“使用物品”逻辑，以改善与后端服务的同步。玩家可能会注意到物品使用之间存在 небольшая задержка。目前预计会出现这种延迟，并且将作为用户体验改进的一部分进一步完善。

- 增加了手动重新同步角色库存的功能。在正常游戏过程中不需要这样做，因为同步会自动进行，但是对于出现意外物品显示行为的情况，存在手动选项，允许玩家尝试自行解决。

- 修复了几个空间音效 (SFX) 问题。

- 额外的错误修复和一般改进。

### 主要后台变更

0.3.6.2中最重大的更新是对所有与库存相关的系统（包括库存、Mentar存储、采集、制作、装备和操作栏）进行的广泛额外工作。

其中一些更改是为了直接准备Reality Thread 7中即将推出的功能，而另一些更改则是由Reality Thread 6期间发现的问题驱动的。

一个被发现的关键问题是，玩家似乎拥有配方所需的正确成分，但配方却失败了。一旦确定，这就需要更深层次的系统级改造，进而影响到许多相关的系统。

### 测试、用户体验变更和持续监控

由于如此多的互连系统受到影响，我们将密切监控我们可能遗漏的任何错误。

这些更新中的大多数在视觉上并不明显，但后端改进为长期开发提供了更强大的支持。

玩家可能会注意到一些行为上的变化，例如：• 移除自动关闭库存用户界面的延迟• 库存用户界面现在立即关闭• 物品同步现在在后台进行

这通过使交互感觉更灵敏来改善整体用户体验，同时同步在后台进行。

在内部，自1月7日以来，团队已经对超过14个单独的版本进行了质量保证测试，因为更改是迭代进行的。 也就是说，鉴于这些变化的规模，我们预计会出现问题，并将紧急解决任何重大问题。

我们现在发布0.3.6.2，以便可以在实时环境中监控这些新的修复和更新，同时团队继续专注于即将发布的Reality Thread 7。

与往常一样，Earth 2团队将继续密切监控性能和玩家反馈，并根据需要进行进一步的改进，以维护和改善E2V1内部的体验。
