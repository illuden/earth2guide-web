> 검토 후 Google Stitch 전달용
> 작성: 2026-04-12 | 업데이트: 2026-04-12

---

## 0. 프로젝트 성격

- **웹앱 (PWA-ready)** — 모바일/PC 동시 지원 반응형
- 모바일 우선 설계 (Mobile First), PC에서 레이아웃 확장
- 브라우저에서 바로 사용, 앱처럼 동작하는 UX 목표
- 설치 가능한 PWA 추후 고려 (manifest.json, service worker)

## 1. 반응형 브레이크포인트

Tailwind CSS 기준:

| 이름 | 너비 | 대상 기기 |
|---|---|---|
| `base` | ~639px | 스마트폰 (기본) |
| `sm` | 640px+ | 큰 스마트폰, 소형 태블릿 |
| `md` | 768px+ | 태블릿 |
| `lg` | 1024px+ | 노트북, 데스크탑 |
| `xl` | 1280px+ | 와이드 모니터 |

## 2. 반응형 레이아웃 변화 규칙

| 모바일 | PC |
|---|---|
| 로고 + 햄버거 메뉴 (☰) | 로고 + 풀 네비 + 검색 + KO/ZH |

| 모바일 | PC |
|---|---|

| 모바일 | PC |
|---|---|
| 전체 너비 단일 컬럼 | 중앙 정렬, max-width 780px, 좌우 여백 |

| 모바일 | PC |
|---|---|
| 사이드바 숨김 → 상단 탭 | 좌측 사이드바 고정 |

## 4. 컴포넌트 목록 (28개)

## 7. 디자인 톤 & 방향성

| 항목 | 방향 |
|---|---|
| 전반적 톤 | 다크 + 미래적 (Earth 2 SF 세계관) |
| 배경 | 다크 네이비 (#0a0f1e 계열) |
| 강조색 | 사이언 블루 / 에메랄드 그린 (#00d4ff, #00ff9d 계열) |
| 폰트 | Geist Sans |
| 카드 스타일 | 글라스모피즘 or 테두리 강조형 |
| 이미지 비율 | 16:9 썸네일 |
| 리퍼럴 배너 | 콘텐츠 방해 없이 자연스럽게 |
| 반응형 원칙 | **Mobile First** — 모바일 기본, PC 확장 |
| 웹앱 형태 | 앱처럼 동작하는 UX, 빠른 전환, 부드러운 인터랙션 |
